Reflection -> 투영된다.
클래스, 메소드, 필드 등의 정보를 분석하여 다루는 기법
객체의 클래스 정보, 클래스 내부의 필드나 메소드에 접근
프레임워크에는 제어의 역전을 구현해놓았다.(사용자가 프레임워크에게 제어를 맡기는 것)
이러한 기능은 프레임워크 등에서 유용하게 활용된다.

 * 리플렉션은 private에 접근할 수 있다. (그 클래스의 구조를 다 뜯어볼 수 있다.)
 * 스프링에서는 리플렉션을 사용해서 런타임 시 등록한 빈을 애플리케이션 내에서 사용할 수 있게 한다.
